# Курсовой проект на тему: Разработка файловой базы данных "Блог"
Выполнила: Маслова Татьяна, бТИИ-251

## Функционал программы
*Приложение для управления структурированными данными о статьях блога.* <br/> 

Поля БД:
- Заголовок статьи
- Автор
- Категория статьи
- Дата публикации
- Кол-во комментариев
- Рейтинг
- Статус модерации<br/>


Возможные операции:
1. Просмотреть все записи – вывод всей базы данных на экран.
2. Добавить новую запись – ввод информации о новом проекте.
3. Поиск статей – одиночный поиск записи по полю «Категория статьи» и комбинированный по полям «Автор» и «Рейтинг».
4. Отсортировать данные – сортировка всей базы данных по рейтингу → дате публикации → категории статьи.
5. Сохранить данные в файл – сохранение имеющихся данных в файл.
6. Загрузить данные из файла – добавление в базу новых данных из файла.
7. Создать тестовые данные – добавление записей со случайными значениями.

## Реализованные функции

### 1. char* get_category(enum Category category)
- Назначение: Получение текстового представления статуса модерации
- Параметры: 
  - category - числовой код категории (тип enum Category)
- Возвращает: указатель на строку-константу (текстовое название категории)

### 2. char* get_status(enum ModerStatus status)
- Назначение: Получение текстового представления статуса модерации
- Параметры: 
  - status - числовой код категории (тип enum ModerStatus)
- Возвращает: указатель на строку-константу (текстовое название статуса)

### 3. int display_blog(struct Blog blog, int record_number)
- Назначение: Обработка элементов массива
- Параметры: 
  - blog - значение структуры Blog
  - record_number - значение структуры Blog
- Возвращает: 1, указывая на успешное выполнение

### 4. int view_all(blogs* blog, int records_count)
- Назначение: Просмотр всех записей
- Параметры: 
  - blog - указатель на массив структур Blog
  - records_count -  счетчик записей
- Возвращает: количество отображенных записей и 0 если записей нет

### 5. int add_blog(blogs* blog, int* records_count, struct Blog* new_blog)
- Назначение: Добавление новой записи
- Параметры: 
  - blog -  указатель на массив структур Blog
  - records_count - указатель на счетчик записей
  - new_blog - указатель на структуру с данными новой записи
- Возвращает: 1 при успешном добавлении, -1 если достигнут лимит записей
### 6. int search_by_range(blogs* blog, int records_count, int search_type, int category_choice, char* search_author, double min_rating)
- Назначение: Поиск записей по категории или по автору+минимальному рейтингу
- Параметры: 
 - blog -  указатель на массив структур Blog
  - records_count - счетчик записей
  - search_type - тип поиска (1 - по категории, 2 - по автору и рейтингу)
  - category_choice - номер категории для поиска (используется при search_type=1)
  - search_author - имя автора для поиска (используется при search_type=2)
  - min_rating - минимальный рейтинг для поиска (используется при search_type=2)
- Возвращает: количество найденных записей, 0 если не найдено, -1/-2 при ошибках ввода
  
### 7. int sort_data(blogs* blog, int records_count)
- Назначение: Сортировка данных по рейтингу, дате и категории
- Параметры: 
  - blog -  указатель на массив структур Blog
  - records_count - счетчик записей
- Возвращает: количество выполненных перестановок элементов (0 если сортировать нечего)
- Примечание: Сортирует по рейтингу (убывание), дате (убывание) и категории (алфавитно)
  
### 8. int save_to_file(blogs* blog, int records_count, char* filename)
- Назначение: Сохранение данных в файл
- Параметры: 
   - blog -  указатель на массив структур Blog
   - records_count - счетчик записей
   - filename - имя файла для сохранения
- Возвращает: количество сохраненных записей, 0 если нечего сохранять, -1 при ошибке файла
  
### 9. int load_from_file(blogs* blog, int* records_count, char* filename, struct Blog* new_blog)
- Назначение: Загрузка данных из файла
- Параметры: 
  - blog -  указатель на массив структур Blog
  - records_count - указатель на счетчик записей
  - filename - имя файла для сохранения 
  - new_blog - указатель на структуру для чтения данных
- Возвращает: количество загруженных записей, -1 при ошибке открытия файла
- Примечание: записи добавляются к существующим, не заменяют их
  
### 10. int create_test(blogs* blog, int* records_count, struct Blog* new_blog, int num_records)
- Назначение: 
- Параметры: 
  - blog -  указатель на массив структур Blog
  - records_count - указатель на счетчик записей
  - new_blog - указатель на структуру для заполнения данными
  - num_records - кол-во создаваемых тестовых данных
- Возвращает: количество созданных записей (-1/-2 при ошибках ввода)
